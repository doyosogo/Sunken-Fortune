# Development Roadmap

## Phase 0: Documentation And Product Foundation

Status: Current phase.

Goals:

- Define product identity.
- Establish legal originality rules.
- Define MVP scope.
- Document economy and monetization constraints.
- Document Autonomous Voyage.
- Document save-system expectations.
- Create a decision log for future changes.

## Phase 1: Prototype Planning

Goals:

- Choose technical stack.
- Define combat prototype requirements.
- Define data model boundaries.
- Draft economy tuning tables.
- Define save payload shape.
- Identify local-first versus backend-later architecture decisions.

Deliverables:

- Technical architecture document.
- Initial combat design specification.
- Initial economy tuning spreadsheet or data file plan.
- Save schema draft.

## Phase 2: Combat Prototype

Goals:

- Build a small playable combat prototype.
- Validate ship movement feel.
- Validate broadside positioning.
- Validate reload timing.
- Validate basic enemy AI.
- Test reward pacing assumptions.

Exit criteria:

- Combat is understandable.
- Combat rewards active decisions.
- Enemy behavior creates pressure.
- Basic progression rewards can be attached without redesigning combat.

## Phase 3: Progression And Economy Prototype

Goals:

- Add gold rewards.
- Add early cannon and ship upgrade progression.
- Add material drops.
- Add basic crafting hooks.
- Add premium currency simulator.
- Add tunable economy values.

Exit criteria:

- Economy can be tuned without code rewrites.
- Gold is the primary progression currency.
- Premium currency has clear earning and spending records.
- Active combat clearly outperforms Autonomous Voyage.

## Phase 4: Save System And Offline Rewards

Goals:

- Implement manual save.
- Implement autosave every 60 seconds.
- Implement major-action saves.
- Implement visible save status.
- Implement Autonomous Voyage calculation.
- Implement Captain's Log / Voyage Report.

Exit criteria:

- Saves are reliable in normal play.
- Offline rewards cap correctly at 14 hours.
- Reward reports clearly show earnings and capped status.
- Economy-impacting actions are recorded safely.

## Phase 5: MVP Content Pass

Goals:

- Add early zones.
- Add enemy variety.
- Add boss prototype.
- Add ship milestone progression.
- Add captain and crew upgrades.
- Improve UI quality.
- Improve balancing.

Exit criteria:

- A new player can complete a coherent early progression loop.
- The first ship milestone feels meaningful.
- Combat, rewards, and save behavior are stable enough for testing.

## Phase 6: Future Expansion Planning

Potential expansion areas:

- PvP.
- Guilds.
- Global chat.
- Co-op encounters.
- Events.
- Cloud save.
- Account system.
- Backend economy validation.
- More bosses and zones.
- Additional ship milestones.

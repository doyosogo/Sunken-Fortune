# Legal Originality Rules

## Purpose

Sunken Fortune is inspired by the broad genre of classic browser pirate MMOs and naval RPGs. Inspiration is acceptable. Copying protected creative expression is not.

These rules exist to keep the project legally original from the beginning.

## Do Not Copy

The project must not copy from existing games:

- Game names.
- Character names.
- Ship names.
- Faction names.
- Region, sea, island, or map names.
- Art assets.
- UI art or exact layouts.
- Icons.
- Music.
- Sound effects.
- Lore.
- Quests.
- Dialogue.
- Item names.
- Enemy names.
- Boss names.
- Exact mechanics presentation.
- Exact economy values.
- Exact progression tables.
- Marketing copy.

## Acceptable Inspiration

The project may use general genre ideas such as:

- Pirate ships.
- Naval combat.
- Cannons and ammunition.
- Ports.
- Treasure.
- Boss encounters.
- PvE zones.
- Crafting.
- Guilds as a future social system.
- Premium currency as a business model pattern.
- Offline progress as a retention mechanic.

These are broad concepts. The specific expression must be original.

## Original Naming Guidelines

All names should be created for Sunken Fortune. Names should fit the world tone but avoid being close variants of known games.

Naming should apply to:

- Ships.
- Bosses.
- Islands.
- Zones.
- Currencies.
- Materials.
- Crew roles.
- Talents.
- Modules.
- Quests.
- Events.

## UI Originality Guidelines

The UI may follow common usability conventions, but should not recreate another game's visual identity.

Allowed:

- Common navigation patterns.
- Standard RPG inventory structures.
- Common combat HUD conventions.
- Clear save indicators.
- Tables, panels, tabs, and menus.

Not allowed:

- Recreating another game's screen composition.
- Matching another game's iconography.
- Reusing color palettes or typography as an identifying style.
- Copying button labels or shop layouts where they are distinctive.

## Art And Audio Guidelines

All art and audio must be:

- Created in-house.
- Licensed for commercial use.
- Generated with rights suitable for the project.
- Properly attributed if attribution is required.

No placeholder from another commercial game should enter the repository.

## Review Process

Before implementing a major feature, review:

| Area | Originality Check |
| --- | --- |
| Naming | Does this name resemble an existing game term too closely? |
| UI | Does this screen look like a direct copy of a known product? |
| Lore | Is this story or faction structure independently created? |
| Economy | Are values designed for this game rather than copied? |
| Assets | Are rights clear and documented? |

## Red Flag Rule

If a feature is being described as "like [specific game] but with minor changes," pause and redesign it at the concept level. Use genre references only to clarify broad intent, never to duplicate execution.

# MVP Scope

## MVP Goal

The MVP should prove that Sunken Fortune is fun and economically coherent as a PvE-first naval action RPG. It should focus on active combat, progression, saving, and a controlled offline reward loop.

## In Scope For MVP

| System | MVP Direction |
| --- | --- |
| Active combat | Core playable experience with movement, positioning, firing, reload timing, and enemy AI. |
| Player ship | Initial ship plus early upgrade path. |
| Cannons | Basic cannon progression and meaningful equipment choices. |
| Ammunition | Standard ammunition and early premium/special ammunition simulation. |
| Gold economy | Main progression currency. |
| Premium currency simulator | Fake premium store with imaginary USD only. |
| Autonomous Voyage | Offline reward system capped at 14 hours. |
| Save system | Manual save button, autosave, major-action saves, visible status. |
| Progression | Captain, crew, equipment, and early crafting hooks. |
| PvE enemies | Basic enemy ship types and early difficulty scaling. |
| Boss prototype | At least one PvE boss pattern eventually, after combat basics are proven. |

## Out Of Scope For MVP

These systems are future expansions:

- PvP.
- Guilds.
- Global chat.
- Co-op.
- Player trading.
- Large-scale social systems.
- Live events.
- Real payment processing.
- Real-money checkout.
- Full MMO backend.
- Mobile-native app builds.
- User-generated content.

## MVP Success Criteria

The MVP should demonstrate:

- Active combat is more rewarding than Autonomous Voyage.
- Manual save and autosave behavior are clear.
- Progression is understandable without social systems.
- Gold remains the main progression currency.
- Premium currency can accelerate but does not erase free progression.
- Autonomous Voyage creates a useful return moment without becoming the game.
- Economy values are easy to tune.

## MVP Risk Boundaries

Avoid building too much before the combat loop is proven. The project should not begin with a full backend, guild system, PvP ladder, or complex marketplace. Those systems depend on progression and economy decisions that should be validated first.
